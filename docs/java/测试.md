# junit测试

```java
// 创建 Mock mock(Class.class) @Mock
// 配置行为
when(mock.method()).thenReturn(value) when(mock.method()).thenThrow(exception) doReturn(value).when(mock).method() doThrow(exception).when(mock).method()
// 参数匹配 any()
anyString(), anyInt(), anyLong() eq(value) argThat(Predicate)
// 验证
verify(mock).method() verify(mock, times(n)).method() verify(mock, never()).method() verify(mock, atLeast(n)).method()
// 捕获参数
ArgumentCaptor<Type> captor = ArgumentCaptor.forClass(Type.class) verify(mock).method(captor.capture()) Type value = captor.getValue()
```

## 单测与集成测试的决策

```plaintext
📊 决策树： 需要测试 Spring 特性（事务、AOP、Security等）？
 ├─ 是 → 使用 @SpringBootTest
	 └─ 否 → 需要测试多个 Bean 的集成？
	 ├─ 是 → 使用 @SpringBootTest + @MockBean
		 └─ 否 → 使用纯 Mockito 单元测试 ✅ (推荐)
```

## 总结

1. **默认使用纯单元测试（Mockito）** - 快速、隔离、易维护
2. **只在必要时使用 @SpringBootTest** - 测试 Spring 特性时
3. **合理使用 Mock** - 只 Mock 外部依赖
4. **遵循 AAA 模式** - Given-When-Then
5. **保持测试独立** - 测试之间不要有依赖
